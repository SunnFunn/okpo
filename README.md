# okpo

Ежедневная и ручная выгрузка реестров вагонов с корпоративной UNC-шары (Windows) на Ubuntu по SFTP.

## Назначение

1. Находит нужный файл реестра в каталоге `Реестры` на файловом сервере.
2. Копирует его во временную папку `tmp/` в корне проекта **с исходным именем**.
3. Передаёт файл на Ubuntu по SSH/SFTP в `remote_dir` **с тем же именем**.
4. Удаляет локальную копию из `tmp/`.

По умолчанию агент работает как демон и каждый день в **07:00 Europe/Moscow** выполняет автопоиск и загрузку.

## Требования

- Windows-машина с доступом к UNC-шаре
- Rust toolchain (`cargo`, `rustc`)
- SSH private key с доступом к Ubuntu-хосту
- На Ubuntu включён SFTP (`Subsystem sftp` в `sshd_config`)

## Конфигурация

Файл [`config.toml`](config.toml) ищется сначала в текущей директории, затем в корне проекта (`CARGO_MANIFEST_DIR`). Если файл не найден — используются значения по умолчанию из кода.

```toml
[source]
base_unc = "\\\\mskfs.rusagrotrans.ru\\Groups\\...\\Реестры"

[schedule]
timezone = "Europe/Moscow"
hour = 7
minute = 0

[ssh]
host = "10.101.139.4"
port = 22
user = "atretyakov"
private_key = "C:\\Users\\tretyakov_av\\.ssh\\id_rsa"
remote_dir = "/home/atretyakov/okpo-agent/data/registers"
```

| Секция | Поле | Описание |
|--------|------|----------|
| `source` | `base_unc` | Корневая папка реестров на шаре |
| `schedule` | `timezone` | Таймзона расписания (IANA) |
| `schedule` | `hour` / `minute` | Время ежедневного запуска |
| `ssh` | `host` / `port` / `user` | Параметры SSH |
| `ssh` | `private_key` | Путь к приватному ключу |
| `ssh` | `remote_dir` | Каталог на Ubuntu для реестров |

## Логика выбора файла (автопоиск)

Время считается в таймзоне из `config.toml` (по умолчанию Москва).

1. Берётся дата **вчера** относительно сегодня.
2. Открывается папка года `YYYY` (например `2026`).
3. Внутри — папка месяца вида `{РусскоеИмя} {YY}` (например `Июль 26`).
4. В **1-й день месяца** вчера попадает в предыдущий месяц (1 января — в предыдущий год).
5. Среди файлов вида `Реестр DD.MM..xls` / `Реестр DD.MM.xls` выбирается файл с **максимальной датой строго меньше сегодняшней**.

Имя на Ubuntu совпадает с исходным именем файла.

## Поведение при существующем файле на Ubuntu

Перед записью проверяется наличие remote-файла. Если он уже есть — в лог пишется предупреждение, файл **перезаписывается**. Это не считается ошибкой.

## Команды

```bat
:: проверка и сборка
cargo check
cargo build
cargo build --release
cargo test

:: справка по флагам
cargo run -- --help

:: демон: ежедневно в 07:00 Europe/Moscow
cargo run --release
.\target\release\okpo.exe

:: разовый автопоиск и загрузка
cargo run -- --once
cargo run --release -- --once
.\target\release\okpo.exe --once

:: ручная загрузка конкретного файла
cargo run -- --file "Реестр 22.07..xls"
cargo run --release -- --file "Реестр 22.07..xls"
.\target\release\okpo.exe --file "Реестр 22.07..xls"
```

Уровень логов можно задать через `RUST_LOG` (по умолчанию `info`):

```bat
set RUST_LOG=debug
cargo run -- --once
```

## Режимы CLI

| Режим | Команда | Поведение |
|-------|---------|-----------|
| Демон | `okpo` | Ждёт расписания, каждый день автопоиск + upload |
| Разовый авто | `okpo --once` | Один прогон с автопоиском |
| Ручная загрузка | `okpo --file "Реестр 22.07..xls"` | Поиск файла по имени на UNC и загрузка |

Флаг `--file` одноразовый: после загрузки процесс завершается. Не комбинируется с `--once`.

## Эксплуатация

- Для срабатывания встроенного таймера процесс должен быть **запущен и не завершён** (сессия пользователя / служба / автозагрузка).
- Альтернатива: Windows Task Scheduler с ежедневным запуском `okpo.exe --once`.
- Временные файлы лежат в `tmp/` в корне проекта и удаляются после **успешной** загрузки на Ubuntu.
