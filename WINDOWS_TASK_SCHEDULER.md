# Запуск okpo через Планировщик заданий Windows

Инструкция для ежедневной выгрузки реестров командой `okpo.exe --once` без постоянно работающего демона. Подходит, если ночью вы блокируете экран (**Win+L** / **Ctrl+L**): блокировка сессию не завершает, задача в 07:00 должна стартовать.

## Зачем планировщик, а не демон `okpo`

| Способ | Плюсы | Минусы |
|--------|-------|--------|
| Демон `okpo` (без флагов) | Не нужен Task Scheduler | Процесс должен висеть; при сне ПК таймер «замирает» |
| `okpo.exe --once` в планировщике | Старт по расписанию ОС; можно будить ПК; не нужен открытый терминал | Нужно один раз настроить задачу |

Рекомендуется именно планировщик.

## Предварительные условия

1. Собрать release-бинарник из корня проекта:

```bat
cargo build --release
```

Исполняемый файл:

```text
<корень_проекта>\target\release\okpo.exe
```

2. В корне проекта должен лежать `config.toml` (пути к шаре, SSH, время в конфиге для демона; для `--once` важны секции `source` и `ssh`).

3. Задача должна выполняться **под вашим доменным пользователем** — тем, у кого есть доступ к UNC-шаре и к SSH-ключу из `config.toml`.

4. Часовой пояс Windows должен быть **Москва** (или время в задаче нужно сдвинуть вручную). Планировщик использует **локальное время Windows**, не поле `schedule` из `config.toml` (оно нужно только демону).

## Рекомендуемая схема путей

Ничего копировать рядом с `src\config.rs` не нужно. Исходники для запуска не требуются.

Удобный вариант — оставить всё в проекте:

```text
C:\Users\tretyakov_av\Apps\okpo\          ← корень проекта = рабочая папка задачи
├── config.toml
├── tmp\                                  ← сюда копируются файлы при работе
└── target\release\okpo.exe               ← программа в действии задачи
```

Подставьте свой реальный путь к репозиторию вместо примера.

В свойствах задачи:

- **Программа:** полный путь к `okpo.exe`
- **Аргументы:** `--once`
- **Рабочая папка (Start in):** корень проекта (там, где `config.toml`)

Так `okpo` найдёт `config.toml` в текущей директории.

## Создание задачи через GUI

1. `Win + R` → введите `taskschd.msc` → Enter.
2. **Библиотека планировщика заданий** → справа **Создать задачу…** (не «Создать простую задачу»).
3. Вкладка **Общие**
   - Имя: `OKPO Register Upload`
   - Описание: `Ежедневная выгрузка реестра на Ubuntu (okpo --once)`
   - Выберите: **Выполнять только для пользователей, вошедших в систему**  
     (при Win+L вы всё ещё «вошедший» — этого достаточно)
   - Параметры безопасности: пользователь — ваш логин
   - «Выполнять с наивысшими правами» обычно **не** включать
4. Вкладка **Триггеры** → **Создать…**
   - Начать задачу: **По расписанию**
   - Ежедневно
   - Время: `07:00:00`
   - Включено: да
5. Вкладка **Действия** → **Создать…**
   - Действие: **Запуск программы**
   - Программа/сценарий:

     ```text
     C:\Users\tretyakov_av\Apps\okpo\target\release\okpo.exe
     ```

   - Добавить аргументы:

     ```text
     --once
     ```

   - Рабочая папка:

     ```text
     C:\Users\tretyakov_av\Apps\okpo
     ```

6. Вкладка **Условия**
   - Включить **Пробуждать компьютер для выполнения этой задачи** (если ПК может уходить в сон)
   - Для ноутбука при необходимости снять «Запускать только при питании от электросети»
   - «Запускать только при подключении к следующей сетевой сети» — обычно выкл.
7. Вкладка **Параметры**
   - Разрешить выполнение задачи по требованию
   - **При пропуске запланированного запуска немедленно выполнить задачу**
   - При сбое перезапускать каждые 10 минут, до 3 раз (по желанию)
   - При желании: останавливать задачу, если выполняется дольше 30 минут
8. ОК → введите пароль пользователя, если Windows запросит.

## Создание задачи через командную строку

Запустите **cmd** от своего пользователя (путь поправьте):

```bat
schtasks /Create /TN "OKPO Register Upload" ^
  /TR "\"C:\Users\tretyakov_av\Apps\okpo\target\release\okpo.exe\" --once" ^
  /SC DAILY /ST 07:00 ^
  /RL LIMITED ^
  /F
```

После этого **обязательно** откройте задачу в `taskschd.msc` и задайте:

- рабочую папку (корневой каталог проекта);
- «Пробуждать компьютер…»;
- при необходимости остальные условия из раздела выше.

`schtasks` сам по себе рабочую папку надёжно не всегда выставляет так, как нужно для `config.toml`.

## Логи в файл

Приложение само пишет логи в `okpo-task.log` в корне проекта (рядом с `config.toml`).
**Каждый запуск перезаписывает файл** — в нём остаются только логи последнего прогона, история не накапливается.

Поэтому в планировщике **не нужно** перенаправлять вывод через `>>` или `>`. Достаточно запускать exe напрямую:

1. Откройте задачу → **Действия** → изменить действие.
2. Программа:

   ```text
   C:\Users\tretyakov_av\Apps\okpo\target\release\okpo.exe
   ```

3. Аргументы:

   ```text
   --once
   ```

4. Рабочая папка — корень проекта (где `config.toml` и появится `okpo-task.log`).

Если раньше в действии был `cmd.exe` с `>> okpo-task.log` — замените на прямой запуск как выше (иначе оболочка снова начнёт дописывать дубликаты в конец файла).

Дополнительно уровень логов:

```bat
set RUST_LOG=debug
```

через переменные среды пользователя или в свойствах задачи (вкладка **Общие** / отдельная обёртка).

Файл `okpo-task.log` уже в `.gitignore` шаблоне проекта (`*.log`).

## Проверка

### Ручной запуск задачи

```bat
schtasks /Run /TN "OKPO Register Upload"
```

Или в `taskschd.msc`: ПКМ по задаче → **Выполнить**.

### Статус и код выхода

```bat
schtasks /Query /TN "OKPO Register Upload" /V /FO LIST
```

В GUI: вкладка **История** (если журнал включён) или смотрите «Результат последнего запуска»:

| Код | Смысл |
|-----|--------|
| `0x0` | Успех |
| другой | Ошибка (смотрите `okpo-task.log` и журнал задачи) |

### Прямой прогон без планировщика

Из корня проекта:

```bat
.\target\release\okpo.exe --once
```

или

```bat
cargo run --release -- --once
```

Если так работает, а задача — нет, почти всегда виноваты **рабочая папка**, **учётная запись** или **сон ПК**.

## Блокировка экрана и сон

| Ситуация | Ожидание |
|----------|----------|
| Win+L / Ctrl+L (экран залочен) | Задача должна выполниться — сессия жива |
| Выход из системы (Log off) | При настройке «только для вошедших» задача **не** выполнится |
| Сон / гибернация | Нужен пункт «Пробуждать компьютер…»; иначе запуск может сдвинуться или пропуститься |
| Полное выключение ПК | Задача не выполнится, пока машина выключена |

Практично для десктопа при питании от сети: в параметрах электропитания отключить сон или разрешить таймеры пробуждения.

## Обновление после правок кода

После изменений в Rust:

```bat
cd /d C:\Users\tretyakov_av\Apps\okpo
cargo build --release
```

Путь к `okpo.exe` в задаче менять не нужно, если он указывает на `target\release\okpo.exe`. Достаточно пересобрать.

Если правили только `config.toml` в корне проекта — пересборка не нужна.

## Удаление задачи

```bat
schtasks /Delete /TN "OKPO Register Upload" /F
```
